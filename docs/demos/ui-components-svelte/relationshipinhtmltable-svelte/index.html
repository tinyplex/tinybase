<RelationshipInHtmlTable /> (Svelte)

In this demo, we showcase the RelationshipInHtmlTable component, a way to display the two Tables linked together by a Relationship.

Rather than rebuilding the whole demo from scratch, we're going back and making changes to the <TableInHtmlTable /> (Svelte) demo again. Where that demo rendered a Table from a Store, this one creates a [Relationships](/api/relationships/interfaces/relationships/relationships/) object and displays one defined Relationship from it.

Set Up

We switch the imports so we can create and provide a [Relationships](/api/relationships/interfaces/relationships/relationships/) object:

Using the RelationshipInHtmlTable Component

The app body can then replace the original table with the joined view, while also customizing the visible columns:

src/App.svelte
<script>
  import {Provider} from 'tinybase/ui-svelte';
  import {RelationshipInHtmlTable} from 'tinybase/ui-svelte-dom';
  import Popularity from './Popularity.svelte';

  let {store, relationships} = $props();

  const customCells = {
    'genres.name': 'Genre',
    'metadata.popularity': {
      label: 'Popularity',
      component: Popularity,
    },
    'metadata.text': 'Description',
  };
</script>

<Provider {store} {relationships}>
  <RelationshipInHtmlTable
    relationshipId="genresMetadata"
    {customCells}
    idColumn={false}
  />
</Provider>

The custom popularity Cell just wraps the current value in bold text:

src/Popularity.svelte
<script>
  import {getCell} from 'tinybase/ui-svelte';

  let {tableId, rowId, cellId, store} = $props();
  const cell = getCell(() => tableId, () => rowId, () => cellId, () => store);
</script>

<b>{cell.current}</b>
